# whereEV3 QA 체크리스트 (안정화·튜닝 단계)

데이터 파이프라인·모바일 앱형 UX 구조는 유지한 채, **재현 가능한 시나리오**와 **알려진 엣지**를 점검한다.

## 환경

- [ ] 모바일: Chrome / Safari (iOS), 뷰포트 ≤900px
- [ ] 데스크톱: Chrome, 뷰포트 >900px
- [ ] 필요 시: 브라우저 뒤로가기·제스처 뒤로가기(모바일 Safari)

---

## A. 모바일 — 히스토리·오버레이

| # | 시나리오 | 기대 동작 | 비고 |
|---|----------|-----------|------|
| A1 | 지도 팝업/목록에서 상세 진입 → 닫기(X) | 상세만 닫힘, 목록 스크롤·선택 충전소 대체로 유지 | `history.back` 1회에 해당 |
| A2 | 상세 연 상태에서 브라우저 뒤로가기 | 상세 닫힘, 스크롤 복원 | `popstate` + 스택 |
| A3 | 필터 열기 → 완료 또는 백드롭 닫기 | 필터만 닫힘 | |
| A4 | 필터 연 상태에서 뒤로가기 | 필터 닫힘 | |
| A5 | 상세 연 채로 목록「지도에서 보기」다른 행 탭 | 상세 닫힘, 선택·지도 반영 | `history.back` 또는 예외 시 수동 정리 |
| A6 | 같은 충전소 상세를 연속으로 다시 열기 | 히스토리 엔트리 중복 없음(동일 세션) | |
| A7 | 상세 A → 상세 B(다른 충전소)로 전환 | 스택 깊이 불필요하게 증가하지 않음(`replaceState`) | |
| A8 | 필터 버튼 매우 빠르게 연타 | `pushState` 1회 수준(가드) | |
| A9 | 상세 연 상태에서 필터 | 필터 버튼 비활성(상세 먼저 닫아야 함) | `[detail,filter]` 히스토리 꼬임 방지 |

### 엣지 / 알려진 한계

- **뷰포트 모바일↔데스크톱 전환**: `overlayStackRef`·플래그는 데스크톱으로 바뀔 때 초기화된다. 잔여 `pushState`는 **전역 `popstate`**로 열린 상세/필터 UI를 맞추는 쪽을 보완했다. 그래도 히스토리 깊이는 브라우저에 남을 수 있다.
- **`pushState` 실패**(극히 드묾): 상세/필터는 열리지만 히스토리 연동 없이 닫기 동작은 일반 `setState` 경로로 처리된다.
- **`history.back()` 예외**: 스택에서 한 번 꺼내고 UI를 수동으로 맞춘다(스택·UI 불일치 완화).

## B. 모바일 — 시트·FAB·모션

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| B1 | 바텀 시트 collapsed ↔ half ↔ full 드래그 | 높이 스냅 자연스럽, 본문 스크롤과 충돌 없음 |
| B2 | 상세/필터 열림 ↔ 닫힘 | Drawer 전환 시간이 과하지 않음(`motion` 토큰) |
| B3 | 상세·필터 닫은 뒤 FAB | 짧은 지연 후 다시 나타남(깜빡임·겹침 최소) |
| B4 | 필터 열린 뒤 지도 탭 | 지도 제스처 차단(레이어·Drawer) |

---

## C. 데스크톱 — 패널·상세

| # | 시나리오 | 기대 동작 |
|---|----------|-----------|
| C1 | 좌측 패널 스크롤 후 지도에서 상세 열기 → 닫기 | 패널 스크롤 위치 복원 |
| C2 | 상세 닫은 뒤 | 지도 선택 마커/선택 상태 유지(`mapSelectedStation`) |
| C3 | 패널 접기/펼치기 | 슬라이드 이징·시간이 다른 모션과 크게 어긋나지 않음 |
| C4 | 데스크톱에서 상세 모달 연 뒤 브라우저 뒤로가기 | 모달이 닫히거나(잔여 히스토리 동기화) 이전 히스토리로 이동 | SPA·동일 출처 전제 |

## 빠른 회귀 (스모크)

1. 모바일: 상세 → 뒤로가기 → 목록 스크롤·행 가시성  
2. 모바일: 필터 → 뒤로가기  
3. 데스크톱: 패널 스크롤 → 상세 → 닫기 → 스크롤 유지  
4. `npm run lint` / `npm run build` 통과
