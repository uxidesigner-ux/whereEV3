# whereEV2 모바일 1차 구조 개편안

현재 코드베이스 기준 문제 진단 및 모바일 MVP 전환을 위한 구조 제안. **코드 수정 전** 검토용 문서.

---

## 1. 현재 문제 진단

### 1.1 왜 모바일에서도 데스크탑 대시보드처럼 느껴지는가

| 원인 | 현재 상태 |
|------|-----------|
| **동일 콘텐츠** | 데스크탑 좌측 패널과 모바일 bottom sheet가 **완전히 같은 `panelBodyContent`** 를 공유함. KPI(StatCard 4개) → 필터 → BarChart → PieChart → footer 순서가 그대로 내려옴. |
| **정보 위계가 “현황” 중심** | 첫 진입 시 제목이 "EV 충전소 **인프라 현황**"이고, 상단이 전체 충전기 수·운영기관 수·충전소 수·충전기 타입 수 등 **집계 KPI**로 시작함. “지금 쓸 충전소 고르기”가 아니라 “인프라를 보여주는 대시보드” 구조. |
| **차트가 1차 정보** | 운영기관별 Top10 BarChart, 충전기 타입 PieChart가 패널 본문 중간에 고정. 모바일에서 시트를 열면 스크롤해야 필터를 쓰고, 그 다음에 차트가 큰 영역을 차지함. “빠른 판단”보다 “분석용 시각화”가 먼저 보임. |
| **필터가 “폼”처럼 배치** | 충전기 타입 Chip + 운영기관/시도/시군구 FilterModalSelect가 KPI·차트와 한 블록에 묶여 있음. “빠른 탐색용 제어”보다 “설정 폼” 느낌. 모바일에서 시트를 열어야만 필터 사용 가능. |
| **진입 시 시트 닫힘** | `mobileSheetOpen` 초기값이 `false`이고 `isMobile` 시 `useEffect`로 닫음. 처음엔 헤더 한 줄만 보이고, “지도만 보는 화면”이 됨. “주변 충전소 목록/요약”이 없어서 닫힌 상태에서 할 수 있는 행동이 거의 없음. |

### 1.2 모바일 UX를 가장 방해하는 영역

- **상단/지도 위 고정 영역 부재**  
  검색·현재 위치·빠른 필터(예: “내 주변” “급속만”)가 지도 위에 없음. 모든 제어가 bottom sheet 안에 있어, 시트를 열지 않으면 “찾기/걸러내기”를 할 수 없음.

- **충전소 목록 없음**  
  테이블/카드/리스트 형태의 “충전소 목록” 컴포넌트가 없음. 지도 마커 + Popup만 있음. “지금 쓸만한 것들”을 리스트로 비교·선택하는 플로우가 불가능함.

- **거리/정렬 미지원**  
  `userLocation`은 지도 뷰 이동과 Circle 표시에만 쓰이고, **거리 기준 정렬·필터**는 없음. “가까운 순” 목록을 줄 수 없음.

- **상세·다음 액션 부족**  
  상세는 Leaflet Popup 고정. “길찾기” “전화” “상세 페이지/시트” 등 **즉시 행동**이 없고, 팝업이 닫히면 정보가 사라짐.

### 1.3 정보 구조·시각적 우선순위·인터랙션 관점

| 관점 | 문제 |
|------|------|
| **정보 구조** | 1순위가 “KPI → 필터 → 차트 → footer”라, “핵심 액션 → 현재 결과 → 필터 → 상세” 위계가 아님. |
| **시각적 우선순위** | 모바일에서 시트를 열면 StatCard 그리드와 차트가 먼저 눈에 들어옴. “지금 보이는 충전소 N개”나 “가까운 순 3곳” 같은 결과가 없음. |
| **인터랙션** | “현재 위치 보기”는 지도 컨트롤에만 있음. “내 주변만 보기” “내 차 타입만” 같은 **한 탭 액션**이 없음. 필터는 시트 내부 모달/드롭다운형이라 단계가 많음. |

## 2. 새 모바일 MVP 핵심 시나리오 3개

다음 시나리오를 1차 개편의 목표 사용 흐름으로 둠.

### 시나리오 A: 지금 내 주변 충전소 찾기

1. 앱 진입 또는 “현재 위치” 터치 → 지도가 내 위치 중심으로 이동, 주변 마커 표시.
2. (선택) 지도 위 또는 시트 헤더 근처 “내 주변” chip/버튼 → **거리 기준 필터/정렬** 적용.
3. bottom sheet에 “현재 보이는 영역 / 내 주변 N곳” **요약 또는 목록**이 보임. 시트를 열지 않아도 “N곳” 정도는 인지 가능하면 좋음.
4. 목록에서 한 건 선택 시 지도 포커스 + 상세(시트 또는 팝업) → 길찾기/전화 등으로 이어짐.

### 시나리오 B: 내 차에 맞는 충전소만 보기

1. 지도 또는 시트 상단의 **충전기 타입(급속/완속 등)** chip/세그먼트로 “내 차에 맞는 타입” 선택.
2. 선택 즉시 지도 마커와 목록이 해당 타입만 표시.
3. (가능하면) “내 차 타입”을 한 번 설정해 두고 유지하는 경로 제공.

### 시나리오 C: 지금 쓸만한 충전소를 빠르게 고르고 상세/길찾기로 이어지기

1. 지도 또는 시트에서 **필터된 결과 중 일부**를 카드/리스트로 표시(가까운 순 등).
2. 한 건 탭 → **충전소 상세** (별도 시트 또는 모달): 이름, 주소, 타입, 이용시간, 전화, “길찾기” “전화걸기” 버튼.
3. 길찾기/전화로 외부 앱 이어지기. 상세를 닫아도 지도/목록 상태 유지.

## 3. 모바일 IA / 화면 구조 제안

### 3.1 모바일 홈 = 지도 메인

- **단일 메인 화면**: 라우터 없이 현재처럼 “지도가 메인” 유지.
- “홈”이 곧 **지도 + 지도 위 컨트롤 + 하단 시트** 구성. 별도 “홈 대시” 페이지는 두지 않음.

### 3.2 지도 메인

- **전체 화면** 유지. 지도 가시 영역 최대 확보.
- **지도 위 고정(또는 플로팅)**  
  - 현재 위치 버튼(기존 LocationControl 유지).  
  - (1차 개편에서 추가) **빠른 필터**: “내 주변” 또는 “급속/완속” 등 최소 개수 chip. 지도 상단 또는 하단 시트 바로 위 작은 영역.
- **마커**: 기존 유지. 클릭 시 Popup 또는 “상세 시트로 이어지는 짧은 팝업” 선택 가능.

### 3.3 Bottom sheet

- **역할**: “현재 결과 요약 + 목록 + 필터 진입”
- **닫힌 상태(collapsed)**  
  - 헤더에 **제목 + “필터된 결과 N곳” 또는 “내 주변 N곳”** 같이 한 줄 요약.  
  - 사용자가 “시트를 열면 목록이 있다”는 것을 인지할 수 있게.
- **연 상태(open)**  
  - 상단: 요약 한 줄(선택) + **빠른 필터 chip**(충전기 타입, “내 주변” 등).  
  - 본문: **충전소 목록**(카드 또는 리스트). 거리순 등 정렬.  
  - 목록 아이템 탭 → 해당 충전소 상세(시트 확장 또는 모달).  
  - 기존 KPI/차트는 **후순위**: 스크롤 아래로 내리거나 “상세/통계” 접기 영역으로.

### 3.4 빠른 필터 영역

- **위치**: 지도 위 플로팅 또는 bottom sheet 헤더/연 상태 상단.
- **구성**:  
  - “내 주변”(거리 기준 필터/정렬),  
  - 충전기 타입 chip(급속/완속 등, 기존 `filterChgerTy` 활용).  
- 운영기관/시도/시군구는 “필터 더 보기”로 시트 내부 또는 모달로 이동(기존 FilterModalSelect 유지 가능).

### 3.5 충전소 상세

- **트리거**: 목록 아이템 탭 또는 마커 Popup에서 “상세 보기”.
- **표시**: 모바일에서는 **시트 확장** 또는 **모달**.  
  - 내용: statNm, 주소, 운영기관, 충전기 타입, 이용시간, 전화.  
  - 액션: **길찾기**(지도 앱/URL), **전화걸기**.
- Popup은 “짧은 요약 + [상세 보기]”만 두고, 본문은 상세 시트/모달에 두는 구분 권장.

### 3.6 하단 meta / footer 정보

- **위치**: bottom sheet **맨 아래** 스크롤 영역(기존 footer와 동일).
- **내용**: “표시 N건”, “전체 약 M건”, “생활안전지도 API”, “© whereevcharger” 등 유지.
- 시트 본문이 “빠른 필터 + 목록” 위주가 되므로, footer는 시각적으로 후순위가 됨.

| BarChart(운영기관별 Top10) | 모바일에서는 시트 하단 접기 섹션 또는 “통계 보기” 열었을 때만 표시. 높이/비중 축소. |
| PieChart(충전기 타입 분포) | 위와 동일. 모바일에서 후순위로. |

### 모바일에서 후순위로 내려야 할 것

| 요소 | 제안 |
|------|------|
| 시도/시군구 필터 | “필터 더 보기” 또는 “상세 필터” 안에 두고, 기본 화면에는 “내 주변” + “충전기 타입”만 노출. |
| 운영기관 필터 | 동일. 필터 더 보기/모달에서 선택. |
| KPI 그리드 전체 | 시트 상단이 “결과 요약 + 빠른 필터 + 목록”이 되도록, KPI는 그 아래 또는 접기 블록으로. |

### 제거하거나 다른 위치로 옮겨야 할 것

| 요소 | 제안 |
|------|------|
| 모바일 시트 본문의 “EV 충전소 인프라 현황” 제목 블록 | 모바일에서는 “주변 충전소” 또는 “검색 결과” 등 **맥락에 맞는 제목**으로 변경. “인프라 현황” 문구는 데스크탑 또는 통계 섹션으로. |
| 모바일에서 시트 열기 전 “할 일 없음” 상태 | 시트 닫힌 상태에서도 “N곳” 요약 표시 + (선택) 지도 위 빠른 필터 chip 추가로, 닫힌 상태에서도 의미 있는 행동 제공. |
| (선택) 패널 본문의 제목 중복 | 모바일 시트 body에서 `'& > * > *:first-of-type': { display: 'none' }` 로 숨기는 대신, **콘텐츠 구조 자체**를 모바일용/데스크탑용으로 분리하면 유지보수에 유리. |

## 5. 1차 개편 범위

- **목표**: “지도 + bottom sheet”에서 **지도 중심 + 시트는 “결과 목록 + 빠른 필터”** 로 전환. 새 기능 대량 추가 없이 **구조와 우선순위만** 조정.
- **범위에 포함**  
  - 모바일 시트 내용 순서/구성 변경(빠른 필터 + 목록 상단, KPI/차트 하단 또는 접기).  
  - 모바일 전용 “충전소 목록” 컴포넌트 추가(필터된 결과를 카드/리스트로 표시).  
  - 지도 위 또는 시트 헤더 근처 “빠른 필터” 영역(충전기 타입 chip, “내 주변” 유도).  
  - 충전소 상세 영역(시트 확장 또는 모달) + 길찾기/전화 버튼.  
  - (데이터 있으면) 거리 기준 정렬/필터 → “내 주변 N곳” 표시.
- **범위에서 제외(1차)**  
  - 라우터 도입, 별도 “홈/상세 페이지” URL.  
  - 즐겨찾기, 내 차 정보 저장 등 신규 기능.  
  - API 변경(현재 Safemap EV API 유지).

## 6. 구현 우선순위

### 1차: 시트 구조 전환 + 목록 + 빠른 필터

1. **모바일 시트 콘텐츠 순서 변경**  
   - `panelBodyContent`를 모바일일 때만 다른 구조로 렌더: 상단에 “결과 N곳” 요약 + **충전기 타입 chip** + (선택) “내 주변” 버튼, 그 다음 **충전소 목록**, 그 아래 기존 StatCard/필터/차트를 접기 또는 한 블록으로.
2. **충전소 목록 컴포넌트 추가**  
   - `filteredItems`를 받아 카드 또는 리스트로 렌더. 모바일 시트 본문에만 사용. 클릭 시 상세로 이어지도록.
3. **빠른 필터 노출**  
   - 충전기 타입 chip을 시트 열렸을 때 상단 고정. (선택) 지도 위 플로팅 chip으로 “급속/완속” 등 한두 개 노출.

### 2차: 상세 + 거리 + “내 주변”

4. **충전소 상세 UI**  
   - 목록/팝업에서 “상세 보기” 시 시트 확장 또는 모달. 주소, 전화, 이용시간, **길찾기** **전화걸기** 버튼.
5. **거리 계산 및 “내 주변”**  
   - `userLocation`이 있으면 `filteredItems`에 거리 필드 추가, 거리순 정렬. “내 주변” chip/버튼 시 거리 제한(예: 5km) 또는 “가까운 순”만 표시.

### 3차: KPI/차트 정리 + 필터 정리

6. **모바일에서 KPI/차트 후순위화**  
   - StatCard를 1줄 요약으로 대체하거나, “통계” 접기 섹션 안으로 이동. BarChart/PieChart도 접기 또는 별도 블록으로.
7. **“필터 더 보기”**  
   - 운영기관/시도/시군구는 버튼 하나로 모달 또는 시트 내 블록 열기. 기존 FilterModalSelect 재사용.
